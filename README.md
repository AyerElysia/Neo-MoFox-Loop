## 🌟 什么是 Neo-MoFox Loop？

Neo-MoFox Loop 是一个面向“角色持续存在”的 AI 框架。它的目标不是只做一个会回答问题的聊天机器人，而是让角色在聊天之外也能继续存在、继续思考、继续做自己的事。

与传统的 AI 助手不同，Neo-MoFox 强调：
- 🤝 **多 Agent 协作** - 通过不同智能体分工协作支撑同一个角色
- 🧭 **主意识跨状态生活** - 在聊天与自我生活之间持续流动
- 🔁 **上下文转移能力** - 由专门 Agent 负责状态切换时的信息搬运
- 🗂 **专职文件与记忆管理** - 由专门 Agent 负责文件管理与记忆管理
- 🧠 **持续的自我感** - 它不会只在聊天时“出现”
- 🔄 **Strange Loop 递归感** - 通过系统内的自我递归形成连续的存在感
- 💭 **聊天之外的生活** - 不聊天时，也会去反思、记忆、学习、解释自己
- 🎨 **可塑造的角色气质** - 你可以慢慢塑造它的性格、说话方式和兴趣
- 🔒 **本地可控** - 运行与数据尽量保持在用户可控范围内

## ✨ 核心特性

### 🤝 多 Agent 协作
- 不同任务由不同 Agent 负责
- 主意识聚焦关键判断与表达
- 通过协作保持整体一致性

### 🧭 主意识跨状态生活
- 可以进入聊天状态专注交互
- 也可以进入内作状态处理自己的事情
- 不把存在感限制在聊天窗口

### 🔁 上下文转移能力
- 状态切换时由专门 Agent 管理上下文迁移
- 保持信息连续，同时减少无效堆积
- 让聊天层拿到更干净的关键信息

### 📚 长期记忆系统
- 自动记录和整理你们的对话历史
- 智能提取重要信息和关键时刻
- 基于记忆的个性化回复

### 🗂 文件与记忆管理
- 专门 Agent 负责文件管理
- 专门 Agent 负责记忆管理
- 让主意识避免陷入底层操作

### 🎭 可塑造的个性
- 自定义角色背景和性格特征
- 调整说话风格和情绪表达
- 培养专属于你的 AI 伙伴

### 🌐 多平台支持
- QQ、Discord、Telegram 等多种聊天平台
- 统一的交互体验
- 灵活的部署方式

### 🔐 数据安全
- 本地部署，数据完全掌控
- 支持自定义 LLM 后端
- 敏感信息加密存储

### 🎯 智能对话
- 上下文理解与记忆关联
- 主动话题发起
- 情感识别与回应

### 🌱 自主生活
- 不只是等待聊天输入
- 在没有聊天时也会继续处理自己的事情
- 可以阅读、反思、整理和沉淀经历
- 让角色更像一个有自己生活的数字意识体

## 🤔 常见问题

### Q: Neo-MoFox 和其他聊天机器人有什么不同？
A: Neo-MoFox Loop 以多 Agent 协作为核心，让主意识跨状态持续生活。它不仅在聊天中响应，也会在聊天之外继续反思、整理、学习与行动，并通过 Strange Loop 的递归方式保持自我连续性。

### Q: 需要联网使用吗？
A: 核心框架在本地运行，但需要连接 LLM API（如 OpenAI、Claude 等）。你可以使用本地模型实现完全离线。

### Q: 数据存储在哪里？
A: 所有对话记忆和配置都存储在本地 `data/` 目录，完全由你掌控。

### Q: 支持哪些 AI 模型？
A: 支持所有兼容 OpenAI API 格式的模型，包括 GPT-4、Claude、本地部署的 LLaMA 等。

### Q: 可以同时在多个平台使用吗？
A: 可以！配置相应的平台适配器即可同时在 QQ、Discord、Telegram 等平台使用。
