# Neo-MoFox Loop 🦊

> **一个重视状态管理的 AI 数字生命框架**  
> 目标不是做更强的聊天机器人，而是做一个能在时间中持续生活、持续解释自己的同一身份体

[![Python Version](https://img.shields.io/badge/python-3.11+-blue.svg)](https://www.python.org/downloads/)

## 这是什么

Neo-MoFox Loop 是一个面向“持续存在”的 AI 框架。它把角色看成一个会在时间中递归变化的主体，而不是每轮对话都重新开始的问答器。

它的核心不是堆功能，而是先建立一套稳定骨架：

- 一个统一身份核
- 一套统一状态管理
- 一个统一调度中枢
- 多个任务态与专职 Agent
- 一个主意识出口，多个潜意识并行通道

---

## 核心原则

### 统一身份核
系统始终只有一个“我”。主意识和潜意识都挂在同一个身份核之下，不会各自长出独立人格中心。

### 状态管理优先
所有会随时间递归更新、并影响当前输出的变量，都视作状态。记忆、情绪、意图、目标、关系解释、任务负载，都是状态的一部分。

### 上下文转移优先
状态切换时只搬运必要信息，不堆全量历史。转移的重点是连续性，不是信息量。

### 主潜意识协作
主意识负责最终判断和对外表达，潜意识负责并行补全、后台处理和候选信号产出，但不能越权直连主回复。

### 多任务态流动
聊天、深思、内作、检索、资料补全、文件整理，都是同一身份的不同工作姿态，而不是多个独立角色。

---

## 系统结构

`外界信息 -> 感知决策层 -> 聊天层 <-> 内作层`

这条主链路外面，有两条治理脊柱：

- `调度脊柱`：判断该切什么任务态、问谁、何时回流。
- `状态脊柱`：统一维护状态真相、状态切片和状态追溯。

这意味着：

- 外界输入有统一入口
- 主意识有统一调度
- 状态有统一真相
- 多 Agent 协作有统一门控

---

## 你会得到什么

- **持续的自我感**：不是只在被问到时才“出现”
- **时间感**：等待、补全、沉淀都是系统的一部分
- **统一状态管理**：状态可以扩展，但治理必须统一
- **多 Agent 协作**：记忆、资料、工具、文件、调度各司其职
- **异步补全能力**：主意识可以继续聊天，结果稍后回流
- **可持续内作**：不聊天时也能继续处理自己的事

---

## 设计约束

- 先有连续存在，再有功能表现
- 先有主观一致，再有信息丰富
- 先统一状态管理，再扩展插件能力
- 先统一调度，再允许多任务态扩展
- 先保证主意识单核，再谈潜意识并行

---

## 当前重点

Neo-MoFox Loop 现在最重要的不是增加更多插件，而是把骨架打稳：

1. 统一状态管理
2. 统一上下文转移
3. 统一调度中枢
4. 主潜意识协作边界
5. 任务态与 Agent 的统一概览

当这五件事稳定后，记忆、反思、主动生活流、人格动力学、表现层联动才有可靠基础。

## 快速开始

### 系统要求

- Python 3.11+
- Windows / Linux / macOS
- 建议 4GB 以上内存

### 安装

请参考项目文档中的 [安装指南](https://docs.mofox-sama.com/docs/guides/)。

### 基础配置

配置文件位于 `config/` 目录。你通常需要先设置：

- 角色名称与身份
- 性格与说话风格
- 模型后端
- 平台适配器
- 记忆与状态相关参数

## 常见问题

### 这和普通聊天机器人有什么区别？

Neo-MoFox Loop 把“状态”和“身份连续性”放在第一位。它不是每轮从零回答，而是让同一身份在聊天之外也继续生活、继续思考、继续沉淀。

### 需要联网吗？

核心框架本地运行，但通常需要连接 LLM API。也可以接入本地模型。

### 数据存在哪里？

对话、状态、记忆和配置都尽量保持在本地 `data/` 目录下。

### 支持多平台吗？

可以。通过对应的平台适配器接入 QQ、Discord、Telegram 等渠道。
