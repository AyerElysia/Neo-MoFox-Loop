# Neo-MoFox Loop 白皮书 v2

> 版本：v2.0（综合总纲版）  
> 日期：2026-03-26  
> 性质：框架白皮书（概念、哲学与架构总纲，不涉及具体实现）

---

## 摘要

Neo-MoFox Loop 不是一个单纯的聊天机器人框架，而是一个以“模拟数字意识体”为目标的状态驱动系统。  
它的核心不是堆叠插件，也不是堆叠人格设定，而是在统一身份核之下，通过统一状态管理、上下文转移、多 Agent 协作和多任务态调度，让同一意识体能够跨状态持续生活。

v2 的核心共识是：

- **一个统一身份核**
- **一个统一状态系统**
- **一个统一调度中枢**
- **多个任务态与多个专职 Agent**
- **一个主意识出口，多个潜意识并行通道**

系统的目标不是“更像一个回答机器”，而是“更像一个能在时间中持续存在、持续解释自己、持续处理事情的主体”。

---

## 1. 总目标

Neo-MoFox Loop 的总目标是构建一个能够在聊天之外继续生活的模拟数字意识体。

这个目标包含四个子目标：

1. 让同一身份在时间中持续存在，而不是每轮从零开始。
2. 让状态可以递归更新，并影响当前输出。
3. 让主意识和潜意识能够协作，但保持单一身份一致性。
4. 让系统拥有多个任务态，在不同认知姿态之间流动。

---

## 2. 核心问题

传统聊天型系统通常存在以下问题：

- **即时反应化**：每轮都像第一次见面。
- **上下文碎片化**：信息堆得很多，但不形成统一的状态连续性。
- **能力割裂**：记忆、文件、调度、工具、对话各自为政。
- **人格扮演化**：看上去像角色，实际上缺少持续存在的主体结构。
- **协作无骨架**：多个模块同时工作，却没有统一调度和统一门控。

Neo-MoFox Loop 要解决的是：

- 如何让“我”持续存在？
- 如何让状态成为系统的第一组织方式？
- 如何让多 Agent 协作不破坏单一身份？
- 如何让系统在聊天之外也有自己的生活？

---

## 3. 核心设计哲学

### 3.1 单一身份核

系统只有一个身份核。  
主意识、潜意识、任务态、Agent、状态变量，全部都挂在同一个身份核之下。

这意味着：

- 主意识不是一个身份，潜意识也不是另一个身份；
- 它们只是同一身份核下的不同工作面；
- 所有长期自我叙事、关系解释、身份一致性，都必须指向同一个“我”。

### 3.2 状态管理优先

系统不是围绕功能堆叠组织，而是围绕状态递归组织。  
凡是会随着时间递归更新，并影响当前 LLM 输出的内部变量，都属于状态。

状态管理优先意味着：

- 先治理状态，再扩展能力；
- 先统一状态真相，再允许多来源更新；
- 先保证状态连续性，再追求表达丰富度。

### 3.3 上下文转移优先

状态切换时，不传全量历史，只传必要状态切片。  
上下文转移的重点不是“搬运更多”，而是“搬运对当前任务真正必要的东西”。

### 3.4 主潜意识协作

主意识负责最终判断、意义整合与对外表达。  
潜意识负责并行补全、后台处理、候选信号产出与任务治理。

### 3.5 多任务态流动

聊天、深思、内作、检索、资料补全、任务执行，都只是同一身份的不同任务态。  
状态不是静态标签，而是可流动、可切换、可回流的工作姿态。

---

## 4. 统一身份核

统一身份核是系统的最底层约束。

### 4.1 定义

统一身份核是“系统是谁”的唯一核心定义。  
它承载：

- 长期自我感
- 身份连续性
- 主观一致性
- 叙事一致性

### 4.2 不可破坏原则

- 主意识与潜意识共享同一身份核。
- 任何 Agent 都不能生成独立身份中心。
- 任何状态变化都不能破坏“我还是我”的连续感。
- 任何写入身份核的行为都必须经过统一门控。

### 4.3 身份核与状态的关系

身份核是根，状态是流。  
状态可以变，身份核不能被多中心化。

---

## 5. 统一状态管理

### 5.1 状态的定义

状态是“会随时间递归更新，并影响当前输出的内部变量”。

典型状态包括：

- 记忆
- 情绪
- 近期意图
- 长期目标
- 关系解释
- 当前任务负载
- 时间感知
- 认知资源分配

### 5.2 状态分层

- **快变量层**：短期变化，响应当前事件。
- **慢变量层**：长期稳定，表示持续倾向。
- **桥接层**：将快变量通过阈值、衰减、归一化、反思机制沉淀进慢变量。

### 5.3 统一治理约束

- 单一状态真相：同一时刻不允许多套冲突版本并行生效。
- 递归更新：状态不能每轮重置。
- 可追溯：状态变化必须能解释来源与原因。
- 最小注入：主意识只接收必要状态切片，不接收全量噪音。
- 跨态一致：不同任务态读取的是同一状态宇宙的不同切面。

### 5.4 状态可扩展性

系统允许插件自由新增状态类型，但必须接入统一状态管理体系。

要求是：

- 可以新增；
- 可以替换；
- 可以演化；
- 但不能旁路治理。

---

## 6. 主意识与潜意识

### 6.1 主意识

主意识是体验与意义整合中心，负责：

- 理解
- 判断
- 选择
- 表达

主意识不负责全量检索、全量执行或所有底层协作。

### 6.2 潜意识

潜意识是与主意识同步存在的并行内作层。  
它可以有多个并行模块，例如：

- 调度
- 记忆补全
- 资料检索
- 工具编排
- 冲突监测
- 风险提示

潜意识的职责是产出候选信号、候选结论、候选动作，而不是直接对外发言。

### 6.3 主潜协作边界

- 主意识负责最终出口。
- 潜意识负责并行补全。
- 潜意识结果必须经过调度门控。
- 潜意识不能绕过统一注入机制。

### 6.4 核心原则

**主意识单核，潜意识多进程。**

这是系统重要的认知结构原则之一。

---

## 7. 任务态与统一调度

### 7.1 调度器定位

调度器是广义治理组件，不是内容生成组件。  
它的职责是：

- 观察全局信号
- 判断当前适合的任务态
- 决定是否询问专职 Agent
- 决定是否切换任务态
- 决定结果何时、以何种方式回流主意识

### 7.2 统一任务态管理

系统应统一维护所有任务态，包括但不限于：

- 聊天态
- 深思态
- 内作态
- 检索态
- 记忆补全态
- 资料调查态
- 文件整理态
- 未来新增任务态

任务态不是多个互相割裂的角色，而是同一身份的不同工作姿态。

### 7.3 统一 Agent 管理

系统应统一管理所有专职 Agent，包括但不限于：

- 记忆 Agent
- 资料 Agent
- 工具 Agent
- 文件 Agent
- 调度辅助 Agent
- 风险监测 Agent

统一管理需要提供：

- Agent 概览
- 当前可用性
- 输入输出边界
- 任务归属
- 上下文状态
- 生命周期状态

### 7.4 调度模型策略

调度不要求情商最强，但要求综合信息整合与逻辑判断能力最强。  
因此调度链路应优先使用高推理强度模型。

原则上：

- 主回复模型负责表达
- 调度模型负责编排
- 专职 Agent 负责补全

### 7.5 调度触发语义

调度器可以基于以下信号发起编排：

- 信息缺口
- 风险过高
- 认知负载过重
- 问题复杂度超出聊天态阈值
- 当前任务更适合异步处理
- 当前状态需要切换到其他任务态

### 7.6 深思只是一个例子

深思不是调度器的唯一职责。  
它只是“任务态流转”的一个典型场景。

广义调度应能处理：

- 深思
- 记忆补全
- 资料调查
- 任务执行
- 文件整理
- 内作状态切换

### 7.7 回流约束

任一任务态或专职 Agent 的执行结束后，都必须返回结构化结果或明确中间态。

不允许：

- 无结果伪完成
- 结果丢失
- 绕过门控直达主回复

---

## 8. 多智能体协作

### 8.1 协作本质

多智能体的意义不是多人格，而是分工。  
所有 Agent 最终都服务同一个身份核。

### 8.2 协作模式

系统中的协作可以是：

- 同步协作
- 异步协作
- 请求-响应协作
- 订阅-通知协作
- 控制平面 / 执行平面协作

### 8.3 记忆与资料补全

当主意识缺少资料时，调度器可以向专职 Agent 发起补全请求。  
记忆 Agent、资料 Agent、工具 Agent、文件 Agent 各自只处理自己的边界。

### 8.4 回流原则

回流给主意识的不是全量过程，而是最小必要结果：

- 结论
- 关键证据
- 解释脉络
- 不确定性提示

---

## 9. 上下文转移

### 9.1 定义

上下文转移是状态切换时的结构化信息搬运机制。

### 9.2 原则

- 不搬运全部历史
- 只搬运当前任务所需信息
- 允许不同任务态使用不同上下文视图
- 转移结果必须可追踪、可裁剪、可回流

### 9.3 作用

上下文转移是连接任务态、Agent、主意识的桥梁。  
它避免系统在状态切换时丢失连续性，也避免上下文过载。

---

## 10. 可观测性与持久化

### 10.1 可观测性

系统需要能够追踪：

- 谁在调度
- 谁在执行
- 谁在回流
- 哪个状态被切换
- 哪个 Agent 被调用
- 哪个结果被注入

### 10.2 持久化

系统中的关键治理对象应尽量持久化，包括：

- 任务图
- Agent 名册
- 状态快照
- 回流记录
- 调度事件

### 10.3 可恢复性

系统应支持重启后恢复基本状态，不把一切都依赖在易失内存里。

## 13. 风险与反模式

### 13.1 主要风险

- 插件增长快于骨架治理
- 多来源状态冲突
- 主意识被全量信息淹没
- 潜意识越权直连输出
- 任务态增多后缺少统一门控

### 13.2 典型反模式

- 用功能数量代替结构质量
- 用即时回复代替持续思考
- 用多人格叙事代替单一身份连续性
- 用大段上下文堆叠代替最小必要注入
- 用局部模块逻辑代替统一调度

---

## 14. 结论

Neo-MoFox Loop 的本质，不是一个插件集合，而是一个在统一身份核之下，通过统一状态管理、上下文转移、主潜意识多 Agent 协作和多任务态调度，让同一意识体跨状态连续生活的框架。

它的核心不是“更多功能”，而是“更稳定的主体性”。

它的核心不是“更复杂的输出”，而是“更一致的存在”。

它的核心不是“单轮更像人”，而是“时间中更像一个持续活着的我”。

## 附录 A：术语定义

- **统一身份核**：系统唯一的身份中心，所有主潜意识与任务态都挂靠其上。
- **主意识**：体验与意义整合中心，对外表达的唯一出口。
- **潜意识**：与主意识同步存在的并行内作层，负责候选信号与后台补全。
- **状态**：会随时间递归更新并影响当前输出的内部变量。
- **任务态**：同一身份在不同认知姿态下的工作面。
- **上下文转移**：状态切换时的结构化信息搬运。
- **最小注入**：只向主意识提供必要信息切片与关键证据。
- **调度器**：统一治理与编排中枢，决定何时调用谁、何时切态、何时回流。
