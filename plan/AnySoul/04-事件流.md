# AnySoul 事件流

> 来源： [事件（Event）| AnySoul 文档](https://docs.anysoul.ai/zh/concepts/event-stream/)

## 核心结论

AnySoul 把 Agent 所面对的外界输入，直接建模成事件流。  
事件不是偶发噪音，而是 Agent 持续生活的一部分。

## 事件是什么

事件可以是：

- 用户消息
- 分享链接
- 定时提醒
- 后台触发
- 前序任务引出的后续事件

AnySoul 强调的不是单条消息，而是连续流动的事件系统。

## 为什么重要

- 没有事件流，Agent 会被动且碎片化
- 有事件流，Agent 才能保持上下文连续
- 事件流能帮助 Agent 判断优先级和保留关键信号

## 对 Neo-MoFox Loop 的启发

- 外界输入应统一进入事件流
- 消息、定时器、后台任务、记忆触发都可视为同类事件
- 调度器可以基于事件流判断要不要切任务态或调用 Agent
