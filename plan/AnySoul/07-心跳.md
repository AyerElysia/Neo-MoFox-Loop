# AnySoul 心跳

> 来源： [心跳 | AnySoul 文档](https://docs.anysoul.ai/zh/concepts/heartbeat/)

## 核心结论

AnySoul 的“心跳”是一种处理循环，也是每日预算单位。  
每次 Agent 处理事件、查看待办、做决策，都会消耗心跳。

## 心跳是什么

心跳可以理解为：

- 一次 Agent 醒来
- 一次处理循环
- 一次思考与行动窗口

它不是单纯的在线时长，而是“做了多少事”的计量。

## 什么会消耗心跳

- 你发送消息
- Agent 自主探索
- 手动唤醒 Agent

## 什么不会消耗心跳

- 只读聊天记录
- 只读记忆
- 被动接收外部事件
- 修改配置

## 心跳与事件、待办

每次心跳，Agent 通常会：

1. 回顾最近事件
2. 检查待办
3. 决定下一步行动

## 预算与休眠

- 心跳每天重置
- 所有 Agent 共享同一个每日池
- 用完后 Agent 休眠
- 数据、记忆、聊天记录都会保留

## 对 Neo-MoFox Loop 的启发

- 可以用类似“周期预算”治理主动性
- 可以把事件流与任务态调度绑定到循环执行
- 可以让 Agent 在空闲时休息，而不是持续空转
