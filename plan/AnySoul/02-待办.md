# AnySoul 待办

> 来源： [核心理念 | AnySoul 文档](https://docs.anysoul.ai/zh/concepts/overview/)

## 内容块原意

待办是执行层，受到 Cursor、Claude Code 等 Plan Mode 的启发。  
它强调的不是“为了拟人而模拟计划”，而是结构化拆解确实更适合长链路任务。

## 关键思想

- 任务需要被拆解、排序、执行、回看。
- 待办不是附属记录，而是执行层本身。
- 结构化计划比临场硬做更适合 LLM 的长任务工作方式。

## 对 Neo-MoFox Loop 的启发

- 可以把待办/任务态作为统一调度的一部分。
- 可以让不同任务态承接不同粒度的工作。
- 可以把长链路目标拆成可回流、可继续、可追踪的阶段。
