# AnySoul 事件

> 来源： [核心理念 | AnySoul 文档](https://docs.anysoul.ai/zh/concepts/overview/)

## 内容块原意

事件是 Agent 需要处理的原始输入流，包括：

- 消息
- 链接
- 观察
- 提醒
- 后续触发

在 AnySoul 里，事件不是临时噪音，而是 Agent 生活流的一部分。

## 关键思想

- Agent 接收的是事件流，而不是单条指令。
- Agent 的处理对象不仅是聊天消息，还包括各种世界输入。
- Agent 拥有独立时间线，因此同一事件在不同 Agent 眼里可以产生不同后续。

## 对 Neo-MoFox Loop 的启发

- 事件可以作为统一输入层，承接外界信息与内部触发。
- 可以把“消息、记忆触发、定时触发、补全触发”统一成事件流。
- 事件流非常适合接入你设想的调度器与多任务态系统。
