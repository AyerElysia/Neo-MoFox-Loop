# AnySoul 浏览器扩展

> 来源： [浏览器扩展 | AnySoul 文档](https://docs.anysoul.ai/zh/guides/browser-extension/)

## 核心结论

AnySoul 的浏览器扩展是一个低权限、显式分享、可定时采集的上下文入口。
它的重点不是“接管浏览器”，而是“让用户控制什么进入记忆和事件流”。

## 扩展的定位

- 不是 Agent 自动驾驶浏览器
- 不是 `<all_urls>` 的广泛抓取器
- 而是白名单式、用户显式触发的内容分享器

## 主要能力

- 手动 Clip
- 选区分享
- 定时分享
- 直播适配
- 模板引擎

## 与 AnySoul 的连接

浏览器扩展通过 webhook ingest 端点把页面内容送入 Agent 事件流。  
它本质上是“页面内容 -> 模板编译 -> JSON payload -> 事件流”的桥梁。

## 为什么值得借鉴

- 低权限
- 显式授权
- 模板化提取
- 定时但不过度打扰
- 适合知识工作者和研究场景

## 对 Neo-MoFox Loop 的启发

- 可以把网页、文档、代码片段做成受控输入源
- 可以把浏览器上下文转成结构化事件，而不是原始长文本
- 可以让外界观察和内部记忆之间有明确边界
