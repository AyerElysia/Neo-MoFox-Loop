# AnySoul 待办规划

> 来源： [待办（Todo）| AnySoul 文档](https://docs.anysoul.ai/zh/concepts/todo-planning/)

## 核心结论

AnySoul 的 Todo 不是为了“让 Agent 像人类记事”，而是为了让长链路执行可行。

## 主要思想

- Todo 受 Plan Mode 启发
- 更适合长任务、拆解任务、持续推进任务
- 适合把目标拆成阶段并跨时间恢复

## 它解决什么

- 拆解目标
- 跟踪中间进度
- 跨时间恢复工作

## Agent 与人的关系

AnySoul 把 Agent 看成可以协作的持续主体：

- 不一定每条消息都即时响应
- 也可能在你不在线时主动联系你
- Todo 是这种协作关系的协调层

## 对 Neo-MoFox Loop 的启发

- Todo/任务态应作为统一调度的一部分
- 长任务应允许跨时间继续推进
- 任务态可以不等同于聊天态，应该是独立的执行姿态
