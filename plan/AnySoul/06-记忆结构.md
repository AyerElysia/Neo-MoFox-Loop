# AnySoul 记忆结构

> 来源： [记忆（Memory）| AnySoul 文档](https://docs.anysoul.ai/zh/concepts/memory-node-edge-lifecycle/)

## 核心结论

AnySoul 的记忆不是静态笔记库，而是一个可生长的工作空间。  
它把记忆落在文件系统上，再通过检索与联想把它激活。

## 为什么是文件系统

文件系统的优势是：

- 可读
- 可检查
- 可组合
- 可按目录组织语义

这让 Agent 的认知更像“看目录 -> 选文件 -> 深入阅读”。

## 三层结构

AnySoul 的 Memory 可以理解成三层：

1. `Storage`：文件与目录是记忆本体
2. `Retrieval`：BM25 + 向量检索 + grep 思路激活记忆
3. `Association`：random walk + 权重连接形成非线性联想

## 联想为什么是概率化的

- 强连接更容易再次被激活
- 弱连接仍有机会出现
- 不会一次性把所有边都塞进上下文

这和真实记忆更接近，也更符合上下文预算约束。

## 对 Neo-MoFox Loop 的启发

- 记忆应是可生长工作空间，不是静态档案
- 记忆可落在文件系统或类似结构化存储中
- 检索 + 联想比全文注入更适合长周期运行
- 强化/弱化/遗忘机制值得直接借鉴
