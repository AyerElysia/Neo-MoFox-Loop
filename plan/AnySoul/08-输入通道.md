# AnySoul 输入通道

> 来源： [Webhook | AnySoul 文档](https://docs.anysoul.ai/zh/guides/event-input-channels/)

## 核心结论

AnySoul 的外部输入不是单一入口，而是多种输入通道共同汇入事件流。
Webhook 是其中最核心的通道之一。

## Webhook 的定位

- 外部系统可以通过 webhook 向 Agent 发送事件
- 每个 Agent 可以有多个 webhook endpoint
- 每个 endpoint 可接不同的集成源

## 关键机制

- 测试模式：先保存 payload，不立即创建事件
- 可视化映射：把外部 payload 映射成标准事件字段
- 鉴权模式：open / api_key / secret
- 敏感字段剥离：避免凭证进入界面或事件流
- 生命周期管理：测试中、已激活、已禁用、轮转

## 事件流意义

Webhook 的价值不只是“接数据”，而是把外界世界变成可治理、可映射、可切换的事件流。

## 对 Neo-MoFox Loop 的启发

- 可以把外界输入统一为事件通道
- 可以用测试模式做映射验证，避免脏数据污染核心状态
- 可以用鉴权、剥离、轮转做通道治理
- 可以把不同来源接入统一调度脊柱
